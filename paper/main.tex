\begin{abstract}
We train three tree-based models (Random Forest, Gradient Boosting, and XGBoost) on the same tabular datasets and compare their SHAP-based and LIME-based global feature importance and \emph{instance-level} (local) stability. Using three public datasets---California Housing, Adult income, and Titanic---we compute pairwise Spearman rank correlation of feature importance (SHAP and LIME), top-$k$ overlap, per-instance stability, and \emph{SHAP--LIME} rank correlation per model. We find that SHAP-derived rankings are often highly correlated across models but local attributions are less stable; LIME rankings can be stable across models yet disagree with SHAP. We report test-set performance and recommend checking consistency across models and across explanation methods (SHAP vs.\ LIME).
\end{abstract}

\noindent\textbf{Keywords:} SHAP; LIME; feature attribution; explainable AI; model comparison; stability; tree ensembles.

\section{Introduction}
\label{sec:intro}

SHAP (SHapley Additive exPlanations) is widely used to explain predictions of machine learning models by attributing importance to each input feature. When multiple models achieve similar predictive performance on the same task, a natural question is whether their explanations agree: do they identify the same features as important, and in the same order?

Prior work has shown that feature attributions can vary with model choice and that different models with comparable accuracy may yield different local explanations. We quantify this phenomenon for \emph{global} SHAP-based feature importance across three popular tree-based models---Random Forest, Gradient Boosting, and XGBoost---on three public datasets. We also address the gap of \emph{instance-level} stability: whether local attributions are stable across models for the same instance, and how to summarize that (e.g., per-instance rank correlation, proportion of ``stable'' instances). We measure agreement via pairwise Spearman rank correlation and top-$k$ overlap. We also compare SHAP with LIME (Local Interpretable Model-agnostic Explanations): both are post-hoc attribution methods, but LIME fits a local linear surrogate while SHAP uses Shapley values; we report whether their global importance rankings agree per model.

\section{Related Work}
\label{sec:related}

Consistency of SHAP within a single model (e.g., tree ensembles) is well studied. The Rashomon set framework addresses explanation variability across models that perform equally well: different models in the set can yield different attributions for the same prediction. SHAP rankings have also been shown to be sensitive to architecture and random initialization in deep learning. We focus on tabular data and tree explainers, and provide a systematic empirical comparison with explicit metrics (Spearman, top-$k$ overlap) that can be replicated.

\section{Methods}
\label{sec:methods}

\subsection{Datasets and Labels}
We use three public datasets:
\begin{itemize}
  \item \textbf{California Housing}: regression; target is median house value (MedHouseVal). Source: sklearn \texttt{fetch\_california\_housing}.
  \item \textbf{Adult}: classification; target is income $>50$K (binary). Source: OpenML adult, version 2. Numeric features only.
  \item \textbf{Titanic}: classification; target is survival (binary). Source: OpenML titanic, version 1.
\end{itemize}
Data are split 80\%/20\% train/test with a fixed random seed (42) for reproducibility.

\subsection{Models}
For each dataset we train three models with default or standard settings ($n\_estimators=100$, fixed \texttt{random\_state}=42):
\begin{itemize}
  \item Random Forest (scikit-learn)
  \item Gradient Boosting (scikit-learn)
  \item XGBoost
\end{itemize}
Regression models are used for California Housing; classifiers for Adult and Titanic.

\subsection{SHAP and Global Importance}
We use TreeExplainer (SHAP) with a small background set (100 samples from the training set) and compute SHAP values for a subsample of the test set (200 samples by default). For each model we define \emph{global} feature importance as the mean absolute SHAP value per feature over this subsample. Features are then ranked by this importance (rank 1 = most important).

\subsection{LIME}
We use LIME (LimeTabularExplainer) on the same subsample of instances. For each instance we fit a local linear surrogate (default 1000 samples) and obtain feature weights; we aggregate to \emph{global} LIME importance as the mean absolute weight per feature and rank features accordingly. LIME is model-agnostic and can yield different rankings than SHAP. We compute LIME-vs-LIME stability (pairwise Spearman and top-$k$ overlap across the three models) and SHAP--LIME rank correlation per model. LIME can be disabled with \texttt{--no-lime} for faster runs.

\subsection{Stability Metrics}
\begin{itemize}
  \item \textbf{Spearman rank correlation}: pairwise, between the feature rankings of each pair of models. High $r$ indicates similar ordering of importance.
  \item \textbf{Top-$k$ overlap (Jaccard)}: for each pair of models, the Jaccard index of the sets of top-$k$ feature indices ($k=5$). Equal to 1 if the same $k$ features are selected (order ignored).
\end{itemize}

\subsection{Instance-level (local) stability}
Global metrics (mean $|$\texttt{SHAP}$|$) do not capture whether \emph{per-instance} attributions agree across models. For each instance in the SHAP subsample we compute: (i) pairwise Spearman correlation of the three models' SHAP vectors (across features), then the mean pairwise $\rho$ over the three pairs; (ii) for each pair, the top-$k$ feature set from that instance's SHAP values, and the Jaccard index between those sets---we take the minimum Jaccard over the three pairs (worst agreement). We summarize across instances by the mean and standard deviation of (i), the \emph{proportion of instances} with mean pairwise $\rho \geq 0.9$ (``Spearman-stable''), the mean of (ii), and the proportion of instances for which all three models have identical top-$k$ sets (``top-$k$-stable''). This addresses the gap of whether local attributions are stable across models for the same instance.

\subsection{Model Performance}
We report test-set metrics to contextualize explanation comparison: RMSE and $R^2$ for regression; accuracy and AUC for classification.

\section{Discussion}
\label{sec:discussion}

Even when model performance is similar (e.g., all three classifiers on Titanic around 78--80\% accuracy), the ordering or composition of the top SHAP features can differ. SHAP and LIME can also disagree on which features matter most, so conclusions should not rely on a single explanation method. Reporting which model and which explainer were used, or checking agreement across models and across methods (SHAP vs.\ LIME), can make conclusions more reliable.

\section{Conclusions}
\label{sec:conclusions}

We quantified SHAP-based and LIME-based global feature importance stability across Random Forest, Gradient Boosting, and XGBoost on three tabular datasets (California Housing, Adult, Titanic). We also evaluated \emph{instance-level} stability: whether local attributions agree across models for the same instance (per-instance Spearman and top-$k$ agreement, and proportion of ``stable'' instances). Pairwise Spearman rank correlations of global feature importance were high (roughly 0.88--0.98) but not perfect; local stability was generally lower, with many instances not Spearman-stable or top-5-stable.

We conclude that a single model's explanation (SHAP or LIME) should not be treated as the only valid answer: which features matter can depend on the model and on the explainer. Practitioners should report which model and which method were used, or check agreement across models and across methods, before drawing strong conclusions.

Future work could include multiple random seeds for confidence intervals and extension to non-tree models.
